## System Architecture
The system follows a client-server architecture where the frontend communicates with a Flask backend responsible for emotion detection, recommendation logic, and data processing.

---

## Workflow
1. User provides mood or text input  
2. Text is preprocessed and vectorized using TF-IDF  
3. Emotion is predicted using a trained Random Forest model  
4. Songs matching the emotion are filtered  
5. Similarity scores rank the recommendations  
6. Results are displayed with playback options  

---

## Model Description

### Dataset

- Tokenization

### Feature Engineering
- TF-IDF feature extraction  

### Model Training
- Supervised training using Random Forest  

### Evaluation Metrics
- Accuracy

---

## Recommendation Logic
Recommendations are generated by combining predicted emotional labels with cosine similarity scores computed from TF-IDF vectors of song lyrics.
